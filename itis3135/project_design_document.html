  6. Page design:
    1. Index Page:
      • Purpose: Landing page for the site.
      • Audience: Anyone who find the site.
      • Content: Basic introductory details.
      • Is there input: No
      • If so, does it need validation: No
      • Buttons/hyperlinks/dropdowns?: No
      • Actions on the page: No
      • Special/other notes: None
    2. About the Van Page:
      • Purpose: Explain the story of the Van.
      • Audience: potential customers.
      • Content: Backstory about the Client and the Van.
      • Is there input: no
      • If so, does it need validation: no
      • Buttons/hyperlinks/dropdowns?: no
      • Actions on the page: no
      • Special/other notes: none
    3. Van Specs Page:
      • Purpose: explain the van's capabilities
      • Audience: customers
      • Content: info about the van's general vapabilities, as well as more details depending on activity.
      • is there input: no
      • if so, does it need validation: no
      • Buttons/hyperlinks/dropdowns?: yes
      • actions on the page: yes
      • special/other notes: will have a dropdown for activity that gives more specific details(ex: changes in cold/snowy climates).
    4. Interior Page:
      • Purpose: show off the interior of the van.
      • Audience: customers
      • Content: information about the interior/living space of the van.
      • is there input: no
      • if so, does it need validation: no
      • Buttons/hyperlinks/dropdowns?: possibly
      • actions on the page: possibly
      • special/other notes: if there are any moving parts (such as how to make the bed), having interactive things like buttons that swap between the options.
    5. How to Rent Page:
      • Purpose: to help people rent the van
      • Audience: customers
      • Content: Shows price to rent, and allows customers to input requests to a table.
      • is there input: yes
      • if so, does it need validation: for some
      • Buttons/hyperlinks/dropdowns?: yes, to see the current renting table.
      • actions on the page: yes, adding to the table
      • special/other notes: allows customers to input data (name, dates to rent, email/phone, etc), and it adds it to a table of existing data from rentals. There is also a way to get the table to see what dates are taken, and what dates are available.
  7. Dynamic Functionality:
    I plan to had dynamic functionality through the use of interactive images that can be swapped through use of a button. I plan to have a dropdown to help show more specific specifications and capabilities of the van. I also plan to have a form and table for the rent page to apply for rental.
