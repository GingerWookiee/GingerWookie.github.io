  6. Page design:
    1. Index Page:
      • Purpose: Landing page for the site.
      • Audience: Anyone who find the site.
      • Content: Basic introductory details.
      • Is there input: No
      • If so, does it need validation: No
      • Buttons/hyperlinks/dropdowns?: No
      • Actions on the page: No
      • Special/other notes: None
    2. About the Van Page:
      • Purpose: Explain the story of the Van.
      • Audience: visitors
      • Content: Backstory about the Client and the Van.
      • Is there input: no
      • If so, does it need validation: no
      • Buttons/hyperlinks/dropdowns?: no
      • Actions on the page: no
      • Special/other notes: none
    3. Van Specs Page:
      • Purpose: explain the van's technical work
      • Audience: visitors
      • Content: info about the van's current vehicular capabilities, such as the engine horsepower, and any plans on upgrades/cahnges for said systems.
      • is there input: no
      • if so, does it need validation: no
      • Buttons/hyperlinks/dropdowns?: yes
      • actions on the page: no
      • special/other notes: will have links to the new parts that the client is planning to use.
    4. Interior Page:
      • Purpose: show off the interior of the van.
      • Audience: customers
      • Content: information about the interior of the van, along with explination of plans for the living space with maybe any drawings/diagrams/renderings of plans.
      • is there input: no
      • if so, does it need validation: no
      • Buttons/hyperlinks/dropdowns?: possibly
      • actions on the page: possibly
      • special/other notes: having interactive things like buttons that swap between the pictures such the empty van from an angle that can swap to a drawing or rendering of the planned look for that area.
    5. Timeline Calculator Page:
      • Purpose: to help calculate the time and cost for the conversion.
      • Audience: client
      • Content: will have a form allowing input for time and cost of different parts of the conversion process, based on what parts the client wants, and it will return the total time in weeks and days, as well as the total price, for the values inputted.
      • is there input: yes
      • if so, does it need validation: for some
      • Buttons/hyperlinks/dropdowns?: yes, to see the current renting table.
      • actions on the page: yes, adding to the table
      • special/other notes: I have ideas to try to implement the time for each part in 2 potential ways. Either via having a blank for weeks and a blank for days for each time, or by having a dropdown to allow input to be interpreted as either days or as weeks.
  7. Dynamic Functionality:
    I plan to had dynamic functionality through the use of interactive images that can be swapped through use of a button. I plan to have a dropdown to help show more specific specifications and capabilities of the van. I also plan to have a form and table for the rent page to apply for rental.
